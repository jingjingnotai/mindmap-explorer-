Overview

What It Does

Generates comprehensive, professional HTML codebase guides with multiple design styles (Editorial, Bauhaus, Neumorphism, Cyberpunk) in under 2 minutes. Helps developers understand 80% of any codebase in 30 minutes through interactive documentation with architecture diagrams, core flows, and onboarding paths.

Tech Stack

  • Language: Markdown (documentation) + HTML/CSS (templates)
  • Framework: Claude Code Skill System
  • Database: None (file-based)
  • Key Libraries: Google Fonts (JetBrains Mono, Inter, Montserrat, Poppins)

Key Metrics

  • Lines of Code: ~3,000 lines (HTML templates + docs)
  • Entry Point: SKILL.md

What Problem Does This Solve

Problem

Developers waste hours reading scattered documentation, exploring unfamiliar codebases, and trying to understand architecture before contributing. Traditional README files don't provide enough structure, and manually creating comprehensive guides is time-consuming.

Targeting User

  • New team members onboarding to existing projects
  • Open source contributors exploring unfamiliar repositories
  • Technical leads documenting system architecture
  • Code reviewers understanding context before reviews

Data Flow

Entry

User invokes skill via /talk-to-codebase command with optional style keyword (cyberpunk, bauhaus, neumorphism)

Processing

  • Skill explores codebase structure (5-6 key files maximum)
  • Reads appropriate template based on style keyword
  • Performs string substitution on placeholders
  • Generates HTML content with architecture diagram

Storage

Writes complete HTML file to {repo-name}-guide.html in current directory

Guarantees

  • Generation completes in under 2 minutes
  • Output is self-contained HTML (no external dependencies except fonts)
  • All styles meet WCAG AA accessibility standards

High-Level Flow

User Command→ Skill Exploration→ Template Rendering→ HTML Generation

Template Layer

  • template.html — Editorial style (default warm dark mode)
  • template-bauhaus.html — Industrial geometric design
  • template-neumorphism.html — Soft UI with shadow depth
  • template-cyberpunk.html — Neon terminal aesthetic

Documentation Layer

  • SKILL.md — Skill workflow and generation rules
  • references/ — Design system catalogs
  • README.md — Usage instructions

Core Flows

Guide Generation Flow

  1. User invokes /talk-to-codebase . cyberpunk style
  2. Skill detects "cyberpunk" keyword → selects template-cyberpunk.html
  3. Quick exploration: Read README, entry point, 2-3 core files (max 5 min)
  4. Read template once from ~/.claude/skills/talk-to-codebase/
  5. Replace all placeholders via string substitution (SERVICE_NAME, ARCHITECTURE_DIAGRAM, etc.)
  6. Write final HTML to talk-to-codebase-guide.html
  7. Notify user to open in browser

Style Selection Flow

  1. Parse user command for style keywords
  2. Match keywords: "cyberpunk" | "neon" → Cyberpunk template
  3. Match keywords: "bauhaus" | "geometric" → Bauhaus template
  4. Match keywords: "neumorphism" | "soft UI" → Neumorphism template
  5. No match → Default to Editorial template

Architecture Diagram Generation Flow

  1. Identify system components from codebase structure
  2. Generate arch-diagram HTML with arch-node + arch-icon elements
  3. Use emoji icons: 📄 (files), 🔍 (explorer), 🎨 (renderer), 📝 (output), 🗂️ (storage)
  4. Arrange in arch-row with arch-arrow separators
  5. Insert into ARCHITECTURE_DIAGRAM placeholder

Data Layer

Storage Model

  • Templates: Self-contained HTML files in ~/.claude/skills/talk-to-codebase/
  • Documentation: Markdown files in repository root and references/
  • Generated Guides: HTML files written to user's working directory

Data Abstractions

No database or persistence layer. All state is captured in template placeholders replaced at generation time.

Integration Points

External Services

  • Google Fonts CDN: Loads JetBrains Mono, Inter, Montserrat, Poppins fonts
  • GitHub URLs: Skill accepts GitHub repository URLs for analysis

Skill System Integration

Invoked via Claude Code skill system. Reads from ~/.claude/skills/ directory and writes to current working directory.

Onboarding Path

Day 1: Understand the Big Picture

  1. Read this guide (30 min)
  2. Read SKILL.md (15 min)
  3. Generate a guide for a sample repo (45 min)

Week 1: Core Domain Knowledge

  1. Understand the template placeholder system
  2. Read all 4 template files to compare design systems
  3. Try generating guides with different styles

Week 2: Contributing

  1. Read references/design-styles-catalog.md for new style ideas
  2. Implement a new template from the catalog
  3. Test accessibility compliance (WCAG AA, keyboard navigation)

Failure Modes

Template Read Failure

If template file is missing or corrupted, skill will error. No fallback template is provided. Users must ensure ~/.claude/skills/talk-to-codebase/ contains valid templates.

Large Codebase Timeout

Skill enforces 5-minute exploration limit. If codebase is too large, may generate incomplete guides. Mitigation: Focus on entry point and core files only.

Invalid HTML Generation

String substitution can produce malformed HTML if placeholder content contains unescaped characters. No validation layer exists. Users should inspect generated HTML before deploying.

Coupling Points

Claude Code Skill System

Tightly coupled to Claude Code's skill invocation and file I/O APIs. Cannot run outside Claude Code environment.

Google Fonts CDN

All templates depend on Google Fonts for typography. If CDN is unreachable, fonts fall back to system defaults (reduced quality).

Template Structure

All templates must maintain identical placeholder names. Changing placeholders in one template requires updating all others.

Scaling & Bottlenecks

Generation Speed

Currently optimized for codebases under 10,000 files. Exploration phase is the bottleneck. Uses heuristics (file names, structure) to avoid reading everything.

Template Size

Each template is ~700-800 lines of HTML/CSS. No minification. Generated guides are ~30KB. Browser rendering handles this easily.

Style Catalog Growth

Currently 4 implemented templates + 15 documented styles in catalog. Adding new styles requires manual HTML/CSS authoring. No programmatic style generation exists.
