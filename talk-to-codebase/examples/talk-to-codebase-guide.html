What It Does

A Claude Code skill that generates professional HTML codebase guides in under 2 minutes, helping developers understand 80% of any codebase in 30 minutes through structured documentation including overview, Principal Engineer's architectural questions, core flows, key files, and onboarding paths. Supports three visual styles: Editorial, Bauhaus, and Neumorphism.

Principal Engineer's Questions

Five critical questions that help you understand the architectural decisions, constraints, and operational characteristics of this system.

What problem does this solve, and for whom?

  • Problem: Developers waste hours understanding unfamiliar codebases, reading hundreds of files without a clear roadmap
  • Users: Software engineers onboarding to new projects, teams documenting microservices, developers maintaining legacy systems
  • Value: Reduces onboarding time from days to hours by providing structured architectural overview and guided reading paths

How does data flow through the system?

  • Entry: User invokes /talk-to-codebase command in Claude Code CLI
  • Processing: Skill reads SKILL.md → explores codebase (5-6 key files) → loads template → performs string substitution
  • Storage: Writes generated HTML file to local filesystem ({repo-name}-guide.html)
  • Guarantees: Synchronous execution, deterministic output, no external dependencies or network calls

Where are the failure modes and how are they handled?

  • Missing Files: No explicit error handling for missing templates or unreadable codebase files (skill assumes files exist)
  • Invalid Input: Gracefully degrades if codebase structure is unclear (uses README + directory structure)
  • Template Errors: String substitution failures would result in placeholders appearing in output HTML
  • Recovery: No retry logic; user must re-run command manually if generation fails

What are the coupling points and can they be reduced?

  • Tight Coupling: SKILL.md is tightly coupled to template placeholders (36 required placeholders must match exactly)
  • Template Independence: Three templates (Editorial, Bauhaus, Neumorphism) are independent — can add/remove styles without affecting others
  • Framework Dependency: Hard dependency on Claude Code skill framework for invocation and file system access
  • Reduction Opportunity: Could introduce validation layer to check template-placeholder compatibility before generation

How does this scale, and what are the bottlenecks?

  • Scaling: Single-user CLI tool with no concurrency requirements; designed for local development use
  • Bottleneck: Limited to reading 5-6 files by design (speed constraint) — larger codebases get approximated views
  • Performance: Target is sub-2-minute generation time; no caching or optimization beyond limiting file reads
  • Constraint: Template file size (~18KB-32KB) loaded on every invocation — acceptable for CLI tool usage pattern

Architecture

Template Layer

  • template.html — Editorial style (dark mode, terracotta/sage accents)
  • template-bauhaus.html — Industrial geometric design (red/yellow/blue primaries)
  • template-neumorphism.html — Soft UI with clay-like tactile interface

Skill Definition Layer

  • SKILL.md — Main skill logic: exploration workflow, template selection, placeholder mapping
  • references/ — Design system documentation (bauhaus-style.md, neumorphism-style.md)

Execution Flow

  • Claude Code harness invokes skill based on user command
  • Skill performs fast exploration (ls, find, selective file reads)
  • Template selection based on user keywords (default: editorial)
  • String substitution for 36+ placeholders
  • Write HTML output to local filesystem

Data Layer

No Persistent Storage

This is a stateless CLI tool with no database or persistent storage. All data flows through:

  • Input: Local filesystem (codebase files read via Claude Code file access)
  • Processing: In-memory string operations (template loading and substitution)
  • Output: Single HTML file written to local filesystem ({repo-name}-guide.html)

Template Storage

  • Templates stored as static HTML files in ~/.claude/skills/talk-to-codebase/
  • No database schema or migrations required
  • Templates are versioned via git (not runtime versioning)

Integration Points

Claude Code Framework

  • Skill Invocation: Registered via SKILL.md frontmatter (name: talk-to-codebase)
  • File System Access: Uses Claude Code's Read and Write tools for file operations
  • Command Parsing: Receives user arguments via skill framework

External Dependencies

  • None: No npm packages, Python libraries, or external services
  • Web Fonts: Templates reference Google Fonts (Poppins, Inter, Montserrat) via CDN
  • Fallback: Font stack includes system fonts as fallback

Onboarding Path

Suggested Reading Order

  1. Start with README.md to understand project purpose
  2. Review main entry point: SKILL.md
  3. Study core request flow and key components
  4. Explore data models and integration points
  5. Run tests and validate local setup

Day 1: Understanding the Workflow

  • Read README.md for installation and quick start
  • Read SKILL.md sections: "When to Use", "Workflow", "Important Rules"
  • Try generating a guide: /talk-to-codebase . neumorphism style
  • Open generated HTML to see output format

Week 1: Exploring Templates

  • Compare all three templates: template.html, template-bauhaus.html, template-neumorphism.html
  • Read design system documentation in references/
  • Understand placeholder substitution system (36 required placeholders)
  • Practice customizing templates by modifying CSS variables

Contributing

  • Test your changes by generating guides for different codebases
  • Submit a PR with examples and documentation updates
